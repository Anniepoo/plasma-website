PLASMA

The Plasma Programming language aims to be a general purpose programming language suited to parallel and concurrent programming. Plasma will use a strong static type system and referential transparency to make it easier to develop and maintain large programs.

Design

Development has started only recently (July 2015) and we're not ready to discuss many design decisions. However some vague details are provided here to get the hype train rolling.

Plasma language:

  • Declaratively pure (referential transparency)
  • Strong static type system supporting:
    • Discriminated unions
    • Algebraic types
    • Other things
  • Straight-forward FFI aimed towards C APIs

Plasma implementation:

  • Compiles to an abstract machine representation
  • Abstract machine has bytecode and textual representations
  • The first version will execute the bytecode kind-of directly (details later)
  • Later versions may compile the bytecode to assembly/machine code and/or LLVM
  • Open source

Progress

We're hard at work

We intend to measure our progress based on these kinds of milestones, in approximate order.

  1. Hello World compiles and executes
  2. Basic runtime: very simple GC
  3. Self hosting ready
  4. Bootstraps
  5. Runtime system improvements
  6. Basic Parallelism and Concurrency
  7. Optimisations
  8. Improved code generation / extra backend targets
  9. Improved Garbage Collection

FAQ

What is it written in?
Mercury, until it can be bootstrapped, then hopefully Plasma
You work on Mercury already, so is this Mercury 2.0?
No. It's more like a compromise between declarative languages and imperative (more familiar) languages. It is not a logic programming language.
Why are you writing it?
A few reasons. The main one is that I feel as if I haven't done anything "cool" computer-science wise since finishing my Ph.D. (Automatic Parallelism for Mercury) and would like to do more of that kind of thing. I also love purely-declarative programming but Haskell and Mercury can be quite difficult to learn. Therefore I'd like to create a compromise, a language that has many of the cool things about declarative programming, without some of the super-abstract or otherwise unfriendly features of these languages. I also have some exciting plans for parallelism and concurrency, but that's a secret.
Why the name Plasma?
It's a secret for now, until then I'll say that I'm referring to the fourth state of matter.
What paradigm is it?
Technically it will be a functional language, however it will look more like a procedural language and be familiar to more people
What is the intended purpose? Web programming / scripting / systems?
I really don't like these separations. A good language with a good implementation should be suitable in many environments.
Will it have a Javascript/Ruby/X backend?
I would prefer to have fewer backends to maintain. Therefore I will concentrate on the two backends that I have planned: bytecode and native code (or LLVM). However, a bytecode intermediate format allows other developers to write bytecode interpreters (or translators) for their favorite environments. eg: the JVM.
